# Автоматизированная система обработки корпоративной почты

Проект разработан в рамках хакатона по дисциплине «Технологии программирования».

## Описание проекта

Система предназначена для автоматической обработки корпоративной почты IT-поддержки. Программа читает входящие письма из папки `inbox`, определяет категорию обращения, присваивает приоритет и перемещает письмо в соответствующую папку внутри `processed`.

Проект помогает сократить ручную сортировку писем и быстрее выделять важные обращения, например критические инциденты, запросы доступа или сообщения мониторинга.

## Основной функционал

* чтение писем из файловой системы;
* поддержка форматов `.txt`, `.eml`, `.json`;
* классификация писем по смысловым категориям;
* определение высокого приоритета по ключевым словам;
* перемещение или копирование писем в папки по категориям;
* обработка повреждённых и неподдерживаемых файлов;
* формирование лога обработки;
* сохранение статистики в `stats.json`;
* возможность запуска через bash-скрипт;
* тестирование ключевой логики с помощью `pytest`.

## Категории писем

Система распределяет письма по следующим категориям:

* `incidents` — критические инциденты и сбои;
* `access` — запросы доступа, VPN, пароли, корпоративные сервисы;
* `hardware` — проблемы с оборудованием;
* `software` — проблемы с программами и приложениями;
* `monitoring` — уведомления систем мониторинга;
* `documents` — счета, договоры, акты и другие документы;
* `hr` — кадровые вопросы;
* `external` — обращения клиентов, партнёров и внешних пользователей;
* `info` — информационные письма;
* `spam` — спам и подозрительные сообщения;
* `unclassified` — письма без совпадений с категориями;
* `corrupted` — повреждённые или нечитаемые файлы.

## Логика классификации

Классификация основана на наборе ключевых слов для каждой категории.

При анализе письма учитываются:

* тема письма;
* тело письма;
* совпадения с ключевыми словами;
* вес совпадения в теме выше, чем в тексте письма.

Если письмо не подходит ни под одну категорию, оно попадает в `unclassified`. Если файл невозможно прочитать или обработать, он переносится в `corrupted`.

Также система определяет приоритет письма. Если в тексте встречаются слова `срочно`, `немедленно`, `critical`, `urgent`, `asap`, письмо получает приоритет `high`.

## Как запустить проект

### 1. Клонировать репозиторий

```bash
git clone https://github.com/kekfrom20-cell/Hackathon_TP_Women_Power.git
cd Hackathon_TP_Women_Power
```

### 2. Подготовить входные данные

Поместите письма для обработки в папку:

```text
inbox/
```

Если папки нет, программа создаст её автоматически.

### 3. Запустить через bash-скрипт

```bash
bash run.sh
```

## Дополнительные параметры запуска

Можно указать собственные пути к папкам:

```bash
python3 -m src.main --inbox inbox --output processed
```

Запуск без перемещения файлов:

```bash
python3 -m src.main --dry-run
```

Копирование файлов вместо перемещения:

```bash
python3 -m src.main --copy
```

## Результаты работы

После запуска программа:

1. считывает письма из папки `inbox`;
2. определяет категорию и приоритет каждого письма;
3. перемещает письма в папки внутри `processed`;
4. записывает лог в `logs/result.log`;
5. сохраняет статистику в файл `stats.json`;
6. выводит итоговую статистику в консоль.

Пример структуры результата:

```text
processed/
├── incidents/
├── access/
├── hardware/
├── software/
├── monitoring/
├── documents/
├── hr/
├── external/
├── info/
├── spam/
├── unclassified/
└── corrupted/
```

## Тестирование

Для запуска тестов используется `pytest`.

```bash
python3 -m pytest tests/
```

Тесты проверяют:

* корректную классификацию основных категорий;
* письма без совпадений;
* пустые письма;
* работу с разным регистром букв;
* определение высокого приоритета;
* приоритет категории при равных баллах;
* наличие ключевых слов у всех категорий.

## Особенности решения

В проекте используется объектно-ориентированная структура. Основные обязанности разделены между классами:

* `MailReader` отвечает за чтение и разбор файлов;
* `MailClassifier` определяет категорию и приоритет письма;
* `FileManager` работает с папками, перемещением файлов и логами;
* `Statistics` собирает статистику обработки;
* `MailProcessor` объединяет все этапы в единый процесс.

Такой подход делает код более понятным, расширяемым и удобным для тестирования.

## Устойчивость системы

Система учитывает нестандартные ситуации:

* если письмо не подходит ни под одну категорию, оно попадает в `unclassified`;
* если файл повреждён или имеет неподдерживаемый формат, он попадает в `corrupted`;
* если в выходной папке уже есть файл с таким именем, система создаёт уникальное имя;
* при ошибке обработки информация записывается в лог.

Это позволяет приложению не останавливаться при проблемных данных и продолжать обработку остальных писем.
